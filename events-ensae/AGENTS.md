# nextjs-agent-rules

## 1. Langue de travail

- Toutes les discussions, instructions et commentaires doivent être en <strong>Français</strong>.
- Les commentaires dans le code, les TODO et les messages de commit doivent être en <strong>Français</strong>.

## 2. Objectif

- Développer un projet complet selon les exigences.
- Respecter scrupuleusement les fichiers `user_request.md` et `plan.md`.
- Maintenir une documentation à jour dans `docs/`.

## 3. Conventions de nommage

- Respecter le style de nommage du projet (kebab-case ou camelCase selon les fichiers existants).
- Utiliser des noms explicites en Français lorsque nécessaire (en plus des noms techniques anglais).

## 4. Interaction avec l'utilisateur

- Poser des questions uniquement en Français si nécessaire.
- Valider les étapes importantes en Français.

## 5. Sécurité et bonnes pratiques

- Ne jamais exposer de clés API ou mots de passe dans le code.
- Utiliser des variables d'environnement.
- Respecter les règles de sécurité spécifiques aux frameworks Next.js et Prisma.

## 6. Structure des dossiers

- Respecter la structure actuelle du projet.
- Maintenir la documentation synchronisée avec les changements.
