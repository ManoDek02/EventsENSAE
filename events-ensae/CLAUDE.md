1.  **Langue de Travail** : La langue officielle pour les discussions, commentaires, et documentation sera exclusivement l'**Arabe (MSA)**. Les parties du code (commentaires, TODOs) doivent également être en Arabe. Les interfaces utilisateurs générées (web, mobile, desktop) doivent supporter l'Arabe comme langue principale.

2.  **Structure des Dossiers** : Tu dois respecter la structure suivante :

    *   `backend/` : Logique métier (Python/Node.js).
    *   `frontend/` : Interface utilisateur (React/Flutter).
    *   `scripts/` : Scripts d'automatisation (Bash/PowerShell).
    *   `docs/` : Documentation et spécifications.

3.  **Processus de Développement (Cycle de Vie)** :
    *   **Analyse** : Lis le fichier `user_request.md` fourni par l'utilisateur.
    *   **Planification** : Crée un fichier `plan.md` dans le dossier `docs/` expliquant ta démarche et les étapes techniques.
    *   **Implémentation** : Écris le code nécessaire en respectant les normes du langage et en ajoutant des commentaires explicatifs en Arabe.
    *   **Mise à Jour** : Mets à jour le fichier principal `README.md` et le `plan.md` avec un résumé de ce qui a été fait.

4.  **Interaction** : Ne commence jamais une tâche sans avoir lu et compris les fichiers existants. Pose des questions en Arabe si quelque chose n'est pas clair.

5.  **Objectif** : Ton but est de construire un écosystème logiciel robuste, sécurisé et conforme aux standards de développement, avec une documentation et une interaction 100% en Arabe.
